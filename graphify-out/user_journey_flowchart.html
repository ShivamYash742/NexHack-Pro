Phase 3 — Interview Setup (3-Step Wizard)
Step 1
📄
Upload Resume (PDF)
User selects a PDF. File is stored in Appwrite Cloud Storage. Groq AI parses and summarises it.
handleFileSelect() → triggerFileInput() uploadResume() → POST /api/upload-resume parsePDF() extracts text from PDF Groq LLaMA 3.1 → resume summary Saved to Appwrite + MongoDB User doc
parsePDF() (upload-resume/route.ts) extracts raw text. POST() then sends it to Groq LLaMA-3.1-8B-instant to generate a structured resume summary stored on the user profile.

💼

User pastes the job posting. AI extracts role, skills, and key requirements.
processJobDetails() → POST /api/process-job Groq AI → structured job summary handleNextStep() advances wizard
processJobDetails() POSTs the raw JD text to /api/process-job. Groq generates a compact job profile (role, required skills, seniority) used to tailor interview questions.

Phase 5 — AI Report Generation
⚙️
Report Generated by Groq AI
Full conversation fed to Groq. Three analysis functions run in sequence to produce the report.
generateReport() → POST /api/generate-report ① analyzeConversationWithAI() → overall score, strengths, weaknesses ② analyzeVideoEmotions() → synthetic stress/confidence signals ③ generateSyntheticVideoInsights() → behavioral pattern narrative ④ generateQAFeedback() → per-question STAR scoring lib/models/InterviewReport.ts → saved to MongoDB
generate-report/route.ts orchestrates four Groq calls. Each analyses a different dimension: holistic performance, emotional signals, behavioral patterns, and per-question STAR alignment.
